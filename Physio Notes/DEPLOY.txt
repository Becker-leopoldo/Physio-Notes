========================================
  PHYSIO NOTES — DEPLOY NO SERVIDOR
========================================

REQUISITOS
----------
- Windows Server
- Python 3.10 ou superior instalado
- Acesso à internet (para instalar dependências e usar as APIs)


PASSO 1 — COPIAR OS ARQUIVOS
-----------------------------
Copie a pasta do projeto para o servidor. Recomenda-se:

  C:\Apps\PhysioNotes\

A estrutura deve ficar assim:

  C:\Apps\PhysioNotes\
  ├── backend\
  │   ├── main.py
  │   ├── database.py
  │   ├── ai.py
  │   ├── transcribe.py
  │   ├── requirements.txt
  │   ├── start.bat
  │   └── .env          ← criar conforme Passo 2
  └── frontend\
      ├── index.html
      ├── manifest.json
      ├── sw.js
      ├── icon-192.png
      └── icon-512.png


PASSO 2 — CRIAR O ARQUIVO .env
--------------------------------
Dentro da pasta backend\, criar o arquivo ".env" com o conteúdo:

  ANTHROPIC_API_KEY=sk-ant-XXXXXXXXXXXXXXXX
  GROQ_API_KEY=gsk_XXXXXXXXXXXXXXXX

Substituir pelos valores reais das chaves de API.


PASSO 3 — INICIAR O SERVIDOR
------------------------------
Abrir o terminal (cmd) na pasta:

  C:\Apps\PhysioNotes\backend\

E executar:

  start.bat

O script vai:
  1. Instalar as dependências automaticamente (pip install)
  2. Subir o servidor na porta configurada (padrão: 8000)

Após iniciado, o sistema estará disponível em:

  http://localhost:8000
  ou
  http://IP_DO_SERVIDOR:8000


ALTERANDO A PORTA
------------------
Para usar uma porta diferente, edite a primeira linha de configuração
do arquivo start.bat:

  set PORT=8000   ← altere o número aqui

Exemplo para rodar na porta 8080:

  set PORT=8080


OBSERVAÇÕES
-----------
- O banco de dados (physio_notes.db) é criado automaticamente
  na primeira execução dentro da pasta backend\
- O frontend já é servido pelo próprio backend — não é necessário
  iniciar nenhum outro serviço
- Para manter o servidor rodando em background, considere configurar
  o start.bat como uma Tarefa Agendada do Windows para iniciar
  automaticamente com o sistema
